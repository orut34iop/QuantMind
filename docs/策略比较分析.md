# TopkDropoutStrategy 与 WeightedStrategy 策略深度解析

基于 `app/utils/recording_strategy.py` 代码实现的深度分析，以下是两种策略的详细差异对比。

## 1. 核心逻辑 (加权机制)

### TopkDropoutStrategy (Top-K 持仓策略)
*   **核心逻辑**: 采用标准的 Top-K 选股逻辑，并结合"缓冲区"机制来控制换手率。它根据预测分数排名选出前 K 只股票。
*   **加权方式**: 通常采用 **等权重 (Equal Weighting)** (即对每只入选股票分配相同的资金)。
*   **缓冲区机制**: 由 `n_drop` 参数控制。即使某只股票的排名略微掉出 Top K，只要它仍在 `topk + n_drop` 的排名范围内，就会继续持有。这种机制避免了因排名微小波动导致的频繁买卖，有效降低了由于换手产生的交易成本。

### WeightedStrategy (信号加权策略)
*   **核心逻辑**: 一种更精细的主动策略，根据预测信心(Score)的强弱按比例分配仓位。
*   **加权方式**: 采用 **基于分数的加权 (Score-Based Weighting)**。预测分数越高的股票，获得的资金分配越多 (`权重 = 分数 / 总分数`)。
*   **进阶风控**:
    *   **`min_score`**: 最低分数阈值。即使某只股票排名很高，如果其绝对分数对应的预期收益过低，也会被过滤掉 (剔除弱信号)。
    *   **`max_weight`**: 单只股票权重上限 (例如 0.1 表示单票持仓不超过 10%)。如果计算出的权重超过此值，多余部分会被截断并重新分配，以避免风险过度集中。

## 2. 参数差异对比表

| 特性维度 | **TopkDropoutStrategy** | **WeightedStrategy** |
| :--- | :--- | :--- |
| **适用优化方法** | **网格搜索 (Grid Search)** (穷举法) | **遗传算法 (Genetic Algorithm)** (进化算法) |
| **核心参数** | `topk`: 目标持仓数量<br>`n_drop`: 缓冲区大小 (控制换手) | `topk`: 最大持仓数量<br>`min_score`: 最低分数阈值<br>`max_weight`: 单票权重上限 |
| **策略目标** | 寻找稳定的 Top K 持仓数量和最佳轮动缓冲 | 寻找信号强度与风险分散之间的最佳平衡点 |
| **持仓类型** | 等权重 (通常) | 信号加权 (分数权重) |

## 3. 场景选择建议

*   **选择 TopkDropoutStrategy 当**:
    *   您希望快速验证因子/模型在大体上是否有效。
    *   您通过简单的“每只股票影响力相同”的组合来观察因子纯粹的排序能力。
    *   您对 **换手率** (交易成本) 非常敏感，希望通过 `n_drop` 专门优化低频轮动。

*   **选择 WeightedStrategy 当**:
    *   您的 Alpha 因子具有很强的区分度 (即排名第1的股票显著优于排名第10的股票)。
    *   您希望通过重仓最强信号来追求更高的 **夏普比率 (Sharpe Ratio)**。
    *   您需要严格的风险控制 (例如，必须强制执行“单票不超过 20%”的风控逻辑)。
