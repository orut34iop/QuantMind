# QuantMind 行情快照写入规范 (V2.0)

**更新日期**: 2026-03-02  
**核心用途**: 账户实时估值 (NAV)、持仓盈亏计算、盘中交易风控闸门拦截。

---

## 1. 业务背景
本项目 (QuantMind Core) 的特征工程已外置。本项目内部消费的行情快照主要服务于：
1.  **Portfolio Service**: 根据最新价计算账户总资产及当前最大回撤。
2.  **Trade Runner**: 在信号执行前，通过最新价校验标的日内跌幅（跌幅过大则拦截买入）。
3.  **Frontend**: 提供基础的现价展示与分钟线渲染。

## 2. 存储规范 (Redis)

*   **Key 格式**: `market:snapshot:{symbol}`
*   **示例**: `market:snapshot:sh600519`, `market:snapshot:sz000001`
*   **数据结构**: Redis **Hash**

### 2.1 强制必填字段 (Required)
为了确保估值准确，以下字段必须存在：

| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `Now` | Float | **最新成交价**。用于计算持仓市值和账户总资产。 | `1500.50` |
| `PreClose` | Float | **昨收价**（复权后）。用于计算当日准确涨跌幅。 | `1495.00` |
| `Open` | Float | 今日开盘价。 | `1490.00` |
| `timestamp` | Integer | **行情产生的 Unix 秒**。系统会根据此字段判定行情新鲜度。 | `1708845000` |

### 2.2 推荐字段 (Optional)
| 字段名 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `High` | Float | 今日最高价。 | `1510.00` |
| `Low` | Float | 今日最低价。 | `1485.00` |
| `Volume` | Integer | 今日累计成交量。 | `1200500` |
| `Amount` | Float | 今日累计成交额。 | `1801250000.0` |

## 3. 写入行为要求

### 3.1 频率与有效期
*   **写入频率**: **每 30 秒** 推送一次全量快照（覆盖更新）。
*   **自动过期 (TTL)**: 建议为每个 Key 设置 **300 秒 (5分钟)** 的过期时间。
    *   *逻辑说明*: 如果外部采集器崩溃，Key 过期消失，`quantmind-runner` 会因为无法获取现价而自动停止所有交易动作（进入安全保护模式）。

### 3.2 批量写入
外部推送器必须使用 **Redis Pipeline** 进行批量 `HSET`，每批次建议 200-500 个标的，以保证写入的原子性。

## 4. 消费端逻辑 (QuantMind 内部行为)

1.  **估值逻辑**: `quantmind-trade` 遍历所有持仓标的，从 Redis 读取 `market:snapshot:{symbol}` 的 `Now` 字段，计算 `市值 = 持仓量 * Now`。
2.  **风控拦截**: 如果 `(Now - PreClose) / PreClose < -0.03`，拦截该标的的任何 BUY 信号。
3.  **新鲜度检查**: 若 `currentTime - timestamp > 60`，系统记录日志警告行情滞后。
