# optimization

用途：优化分析相关功能。

## 说明
- 归属路径：electron\src\components\optimization
- 修改本目录代码后请同步更新本 README

## 近期更新
- 参数优化排队状态展示（2026-03-19）：
  - 参数优化历史与详情面板新增 `pending/queued` 状态展示，分别显示“待启动/排队中”。
  - 当全局执行槽位已被占用时，新提交任务会在前端明确提示“排队中”，避免被误认为失败或已取消。
  - 排队提示进一步细化为“前方还有 N 个任务”，主进度摘要与历史详情会优先显示当前排队深度。
- 参数优化组合数文案澄清（2026-03-19）：
  - `GridSearchPanel` 顶部说明改为明确区分“系统上限 40 组”与“当前任务实际评估组数”。
  - `ParameterGrid` 统计卡片将“参数组合数”改为“本次预计评估组数”，并增加“前后端同时限制 40 组”的提示，避免将单次任务的 25 组误读为系统阈值。
- 参数优化结果基线与进度口径修复（2026-03-19）：
  - `OptimizationResults` 不再把“相对基线提升”回退到任意首个结果；只有本轮真实命中基线组合时才计算提升，并展示基线参数与对应指标值。
  - 若本轮搜索区间未包含基线组合，会明确提示“无可比基线”，避免出现没有参照物却显示百分比提升的误导。
  - `GridSearchPanel` 与 `OptimizationProgress` 改为共享同一套进度汇总口径，顶部摘要条和下方详情卡片统一按已评估数/总任务数显示，修复轮询状态与任务列表不同步问题。
- 参数优化停止链路补强（2026-03-10）：
  - `GridSearchPanel` 停止按钮优先使用 `currentTaskId`，缺失时回退 `selectedDetail.task_id`，避免页面恢复/切换后无法停止后端任务。
  - 停止后详情刷新优先使用 `currentOptimizationId`，否则回退当前选中优化记录，确保状态能及时展示为 `cancelled`。
- 参数优化右侧历史面板可折叠（2026-03-10）：
  - Grid Search 页面新增“显示/隐藏优化历史”切换按钮。
  - 右侧“优化历史/历史详情”面板默认收起，按需展开，减少主配置区干扰。
- 参数优化进度与历史恢复（2026-03-10）：
  - Grid Search 页面改为“当前优化主面板 + 右侧历史侧栏”布局，历史记录支持 `running/completed/failed/cancelled`。
  - 当前运行中的优化会按后端 `completed_count/failed_count/current_params` 渲染真实任务状态，不再出现“总进度推进但已完成仍为 0”的失真展示。
  - 新增历史详情恢复：离开页面后重新进入参数优化模块，会自动恢复最近的运行中任务，并继续轮询进度与日志。
  - 历史记录中的已完成任务支持一键回填最优参数到快速回测。
- 参数优化调用统一回测服务（backtestService.optimizeQlibParameters / runBacktest）
- 参数优化前端支持异步任务进度显示（轮询任务状态）
- 参数优化实验室“使用指南”弹窗已同步收敛：仅展示 Grid Search 说明，移除遗传算法对比文案，避免与当前能力不一致。
- 参数优化入口调整为“Beta 受限可用版”：
  - 仅保留 TopkDropout 网格搜索（暂不开放遗传算法入口）。
  - 参数范围增加安全约束：`topk<=80`、`n_drop<=20`、组合数 `<=40`。
  - 优化目标收敛为 `夏普比率 / 年化收益率` 两项。
  - 启动前新增 `/api/v1/qlib/health` 检测，`redis_ok` 不可用时禁用开始按钮并提示。
  - 结果区简化为“最优参数 + 目标值 + 相对基线提升”，支持“一键回填”到快速回测配置，并自动切回快速回测模块。
- 参数优化身份与任务控制修复（2026-03-10）：
  - 优化请求不再硬编码 `user_id='optimization'`，统一改为当前登录用户身份，避免引擎侧返回 `403 user_id does not match authenticated identity`。
  - “停止优化”按钮改为真正调用后端 `/task/{task_id}/stop`，不再只是前端本地中断轮询。
  - 遗传优化结果区补齐“一键回填”能力，和网格搜索保持一致。
